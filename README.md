# Replay Master

**Превращайте киберспортовые повторы StarCraft II в кинематографичные 3D-ролики за пару кликов.**

![Platform](https://img.shields.io/badge/OS-Windows%2010%2F11-blue)
![Blender](https://img.shields.io/badge/Blender-4.4%2B-orange)
![Python](https://img.shields.io/badge/Python-3.10%2B-green)
![License](https://img.shields.io/badge/License-MIT-yellow)

**Replay Master** — это десктопное приложение-оркестратор, которое автоматизирует рутину по созданию 3D-синематиков из RTS. Вы загружаете обычный `.SC2Replay`, а программа сама находит самое эпичное сражение, строит 3D-сцену с тысячами юнитов, настраивает кинематографическую камеру и рендерит готовое видео.

### Для кого этот инструмент?
*   **Gamedev-разработчики и Cinematic-художники:** Быстрый прототипинг баттл-сцен, создание пре-визуализации (pre-vis) для собственных проектов.
*   **Создатели машинимы и контента:** Генерация динамичных вставок для разборов матчей, YouTube-роликов и стримов.
*   **Аналитики и фанаты:** Визуализация тактических маневров и "вау-моментов" в высоком качестве.

## ️ Критические требования

Прежде чем начать, убедитесь, что ваша система соответствует строгим требованиям движка:

1.  **ОС:** Только **Windows 10 / 11** (64-bit).
2.  **Blender:** Строго версии **4.4 или выше**. 
    > *Почему?* Скрипт генерации сцены использует новый API *Layered Actions* для работы с анимациями, который появился только в 4.4. На более старых версиях анимации юнитов будут работать некорректно или вызывать краш.
3.  **Ассеты:** Наличие экспортированных моделей юнитов, зданий и текстур StarCraft II в формате `.blend` и `.dds/.png`.

## Быстрый старт (Для пользователей .exe)

Если вы скачали готовую сборку `.exe`, настройка займет минуту:

1.  **Установите Blender 4.4+** с официального сайта. При установке убедитесь, что он сохраняется в стандартную директорию (`C:\Program Files\Blender Foundation\...`), либо добавьте путь к `blender.exe` в переменные среды Windows (PATH).
2.  **Подготовьте ассеты.** Создайте три папки: `units`, `buildings`, `textures` и поместите туда соответствующие файлы (см. раздел *Подготовка ассетов* ниже).
3.  **Запустите `Replay_Master.exe`**.
4.  **Мастер настройки:**
    *   **Шаг 1:** Укажите путь к `.SC2Replay` и папку, куда сохранить итоговое видео. Настройте длительность ролика (30-75 сек).
    *   **Шаг 2:** Укажите пути к папкам с моделями и текстурами (или доверьтесь автопоиску).
    *   **Шаг 3:** Нажмите **«Сгенерировать»** и пейте кофе. Приложение само всё распарсит, найдёт главную битву, построит сцену и отрендерит видео.

## Запуск из исходного кода (Для разработчиков)

Если вы хотите модифицировать логику, добавить поддержку новых игр или отлаживать 3D-сцену вручную.

### 1. Установка окружения
```bash
git clone <repository_url>
cd ReplayMaster
pip install -r requirements.txt
```
*(Основные зависимости: `sc2reader`, `opencv-python`, `Pillow`)*

### 2. Запуск GUI
Файл `frontend_main.py` находится в папке `frontend/`. 
Запускать нужно либо через корневой файл-обертку (для корректного разрешения путей), либо напрямую:
```bash
python frontend/frontend_main.py
```

### 3. Ручная отладка 3D-сцены (Blender)
Вам не нужно запускать весь пайплайн, чтобы проверить, как Blender читает конкретный JSON.
1. Откройте Blender 4.4+.
2. Перейдите во вкладку **Scripting**.
3. Откройте `import_to_blender.py`.
4. В **Разделе 1** (в начале файла) заполните моковые пути к вашим JSON и ассетам.
5. Нажмите **Run Script**. Сцена построится прямо в viewport.

## Подготовка ассетов (Важно!)

StarCraft II хранит модели в проприетарном формате `.m3`. Replay Master работает с `.blend`. 
Для первичной конвертации моделей используйте встроенную утилиту `parce_m3_models.py`:

1. Откройте Blender.
2. Во вкладке *Scripting* откройте `parce_m3_models.py`.
3. Укажите `SOURCE_DIR` (папка с извлеченными `.m3` файлами) и `DEST_DIR` (куда сохранить `.blend`).
4. Запустите скрипт. Он пакетно импортирует модели,.extracted анимации и сохранит их в удобном формате.

*Совет: Для извлечения `.m3` файлов из модов SC2 рекомендуется использовать сторонние утилиты вроде `Cascade` или `CASCExplorer`.*

## ️ Как это работает? (Под капотом)

Приложение использует 4-ступенчатый пайплайн:

1.  **Парсинг (`parce_replay.py`):** Библиотека `sc2reader` извлекает из бинарного реплея миллионы событий. Скрипт фильтрует "шум" (микро-движения), центрирует координаты карты относительно `(0,0)` и сохраняет всё в единый `unified.json`.
2.  **Аналитика (`find_main_battle.py`):** Алгоритм ищет "эпицентр" матча. Используется метод скользящего окна (30 секунд) и пространственная фильтрация, чтобы найти интервал с максимальной плотностью смертей боевых юнитов.
3.  **Оркестрация (`controller_backend.py`):** Бэкенд перехватывает `stdout`, формирует `render_payload.json` и запускает Blender в headless-режиме (без GUI) через `subprocess`.
4.  **3D-Движок (`import_to_blender.py`):** Сердце проекта. Blender читает JSON, дублирует ассеты из кэша, применяет математику поворотов (Shortest Path), настраивает NLA-треки для анимаций, создает процедурные материалы и запускает кинематографическую камеру на орбите вокруг центра битвы.

## Протокол взаимодействия (IPC)

Поскольку Blender работает как отдельный процесс, обмен прогрессом идет через `stdout`. 
Если вы пишете свои скрипты для этого проекта, используйте стандартный вывод для связи с UI:

*   **Прогресс:** `print('__UI__ {"stage": "import_scene", "progress": 45, "message": "Загрузка..."}')`
*   **Готовность превью:** `print('__GLB__')` (Сигнал фронту подгрузить 3D-превью сцены).

## Troubleshooting (Частые проблемы)

| Проблема | Решение |
| :--- | :--- |
| **Blender не найден / Ошибка запуска** | Убедитесь, что Blender 4.4+ установлен. Либо добавьте путь к `blender.exe` в системную переменную `PATH`, либо задайте её вручную через переменную окружения `BLENDER_EXE`. |
| **Анимации юнитов дергаются или отсутствуют** | Вы используете Blender 4.3 или ниже. Обновите Blender до 4.4+ (это строгое требование нового API). |
| **Видео черное / Нет моделей** | Проверьте пути к ассетам. Папки должны содержать подстроки `unit`, `building`, `texture` в названиях, чтобы автопоиск их нашел. |
| **Не найдено главное сражение** | В реплее было мало смертей (например, глухая оборона или ранний GG). Алгоритм всё равно найдет окно с *максимальным* количеством смертей, но оно может быть не таким зрелищным. |

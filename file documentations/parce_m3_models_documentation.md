# Документация: `parce_m3_models.py`

## Назначение
Пакетный конвертер 3D-моделей зданий StarCraft II из проприетарного формата `.m3` в формат `.blend` (Blender). 

## Описание
Это **автономный (standalone) скрипт-утилита**, который не интегрирован в основной пайплайн проекта. Он запускается вручную разработчиком (или через CI/CD) внутри среды Blender, когда необходимо подготовить библиотеку 3D-ассетов. Скрипт рекурсивно обходит указанную директорию, импортирует найденные `.m3` файлы, корректно извлекает и сохраняет их анимации, а затем экспортирует результат в `.blend`.

---

## Структура (High-Level Overview)

В файле нет классов, логика реализована через функции и основной блок выполнения.

### 1. Функции обработки
*   **`clean_scene()`**: Критически важная функция "гигиены". Полностью очищает сцену Blender, удаляет неиспользуемые действия (анимации), чистит орфанные данные и вызывает `gc.collect()`. Предотвращает утечки памяти и пересечение данных между разными моделями при пакетной обработке.
*   **`process_m3_file(m3_path, blend_path)`**: Ядро конвертации. 
    *   Импортирует `.m3` (используя `getattr(bpy.ops.m3, 'import')`, так как `import` — зарезервированное слово).
    *   Ищет арматуру (скелет) и итерирует `m3_animation_groups`.
    *   Переименовывает анимации в формат `M3_{group}_{subanim}` и помечает их как `Fake User` (чтобы Blender не удалил их при сохранении).
    *   Сохраняет итоговый `.blend` с компрессией.

### 2. Главный цикл (Main Block)
*   Настраивает логирование (в файл и консоль).
*   Рекурсивно обходит `SOURCE_DIR` через `os.walk`.
*   Проверяет наличие уже сконвертированных `.blend` файлов (пропускает их для поддержки прерванных запусков).
*   Запускает обработку, ведет подсчет успешных/ошибочных конвертаций и замеряет общее время.

## ️ Важные технические детали (Для разработчиков)

Если вы планируете запускать или модифицировать этот скрипт, учтите следующие нюансы:

1.  **Среда выполнения (Критично!)**
    Скрипт использует библиотеку `bpy` (Blender Python API). Его **нельзя** запустить через стандартный интерпретатор Python (`python parce_m3_models.py`). Он должен выполняться либо как скрипт внутри запущенного Blender, либо через headless-режим Blender (`blender --background --python parce_m3_models.py`).
2.  **Зависимость от стороннего аддона**
    Строка `getattr(bpy.ops.m3, 'import')` подразумевает, что в Blender заранее установлен и включен аддон для работы с M3 (например, *io_scene_m3*). Без него скрипт упадет с ошибкой `AttributeError`.
3.  **Отсутствующая константа `DEST_DIR`**
    В блоке настроек объявлен только `SOURCE_DIR`, но в главном цикле используется `Path(DEST_DIR)`. **В текущем виде скрипт упадет с `NameError`**. Перед запуском необходимо добавить определение `DEST_DIR` в секцию `================= НАСТРОЙКИ =================`.
4.  **Специфика сохранения анимаций**
    Логика в `process_m3_file` жестко завязана на внутренние свойства аддона M3 (`obj.m3_animation_groups`). Если аддон обновится и изменит API, парсинг анимаций сломается. Также стоит обратить внимание, что скрипт берет только *первый* найденный `.m3` файл в каждой подпапке (`m3s[0]`).
5.  **Защита от прерываний**
    Скрипт умеет "догонять" прерванный процесс: если `.blend` файл уже существует в папке назначения, он пропускается (`if os.path.exists(blend_path): continue`). Это позволяет безопасно перезапускать его на больших объемах данных.
